1. Django’s cache framework

A fundamental trade-off in dynamic websites is, well, they’re dynamic. Each time a user requests a page, the Web server makes all sorts of calculations – from database queries to template rendering to business logic – to create the page that your site’s visitor sees. This is a lot more expensive, from a processing-overhead perspective, than your standard read-a-file-off-the-filesystem server arrangement.

That’s where caching comes in.

3. Installation

Type below command in CMD.you should have preinstalled pip in your system.
pip install django

4. Start New Web Project

Django project can contain number of apps inside for different purpose.

Navigate to your directory to create project and execute below command to create project.

django-admin startproject < project_name >

django-admin startproject mysite

you can see below folder structure created inside folder "mysite"

To run the Project

cd mysite python manage.py runserver

Now server is running, visit: http://127.0.0.1:8000


You can change port by

python manage.py runserver 8000

If you want to change the server’s IP, pass it along with the port.

python manage.py runserver 0.0.0.0:8000
